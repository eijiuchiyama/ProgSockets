\section{Protocolo de comunicação}

Foi utilizado como protocolo da camada de aplicação o Distributed Resource Control Protocol (DRCP), criado por nós mesmos.

\subsection{Métodos}

\begin{itemize}
	\item CREATE(valor string/int) \(->\) id int \(|\) error
	\subitem Caso sucesso: id do recurso criado (int)
	\subitem Casos erro: Limite de recursos (5), Valor inválido (6)
	
	\item GET(id int) \(->\) valor string/int \(|\) error
	\subitem Caso sucesso: valor string/int
	\subitem Casos de erro: Recurso já reservado por alguém (1), Recurso inexistente (2), Recurso livre porém não reservado (3)
	
	\item SET(id int, valor string/int) \(->\) ok \(|\) error
	\subitem Caso sucesso: ok
	\subitem Casos de erro: Recurso já reservado por alguém (1), Recurso inexistente (2), Recurso livre porém não reservado (3), Valor inválido (6)
	
	\item RESERVE(id int) \(->\) ok \(|\) error
	\subitem Caso sucesso: ok
	\subitem Casos erro: Recurso já reservado por alguém (1), Recurso inexistente (2)
	
	\item - RELEASE(id int) \(->\) ok \(|\) error
	\subitem Caso sucesso: ok
	\subitem Casos erro: Recurso já reservado por alguém (1), Recurso inexistente (2), Recurso livre porém não reservado (3)
	
	\item - LIST() \(->\) ok \(|\) error
	\subitem Caso sucesso: count int, lista de ids (Recurso[])
	\subitem Casos de erro: nenhum
\end{itemize}

\subsection{Códigos de erro}

\begin{itemize}
	\item Recurso já reservado por alguém (1)
	\item Recurso inexistente (2)
	\item Recurso livre porém não reservado (3)
	\item Limite de recursos (5)
	\item Valor inválido (6)
	\item Método inexistente (7)
	\item Bad request (8)
	\item Internal server error (500)
\end{itemize}

\section{Estrutura de dados do servidor}

Os dados no servidor são armazenados em um vetor de Resources que, por sua vez, é uma estrutura criada por nós e apresentada abaixo:

\begin{Verbatim}[tabsize=4]
typedef struct{
	int id;
	char* value;
	bool reserved;
	pthread_t* belongs_to;
} Resource;
\end{Verbatim}
